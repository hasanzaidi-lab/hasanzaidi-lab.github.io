Tech Stack & Where I Use It

UI & App
SwiftSwiftUIUIKitCombine

Used SwiftUI for new feature surfaces (Flight Tracker UI), UIKit for complex & legacy banking screens, Combine for event streams and synchronization.

Storage & Sync
Core DataCloudKitKeychain

Core Data for local persistence (offline-first flows), CloudKit for shared lists, Keychain for auth tokens and secure data.

Networking
URLSessionAlamofireGraphQL

Prefer structured concurrency + URLSession for predictable flows; use Alamofire for complex multipart/uploads and compatibility layers.

Infra & Testing
GitHub ActionsFastlaneSnapshotTesting

CI for build and test automation, Fastlane for releases, SnapshotTesting + XCTest for UI regression and unit tests.

Libraries & Tools (examples)

Alamofire (network helpers), Kingfisher (image caching), Realm (prototyping alternative to Core Data), SwiftyJSON (parsing), CocoaLumberjack / XCGLogger (logging), Charts / Recharts (data visualizations), Sentry/Datadog for monitoring.

How I apply them

I modularize features into frameworks (UI + domain + data). I keep networking layer decoupled for testability, and abstract persistence behind protocols to allow swapping Core Data with Realm for prototypes.

Process

Feature-driven development, reviews with pre-merge CI gates, staged releases via Fastlane/TestFlight, and post-release monitoring with Datadog & Sentry.
